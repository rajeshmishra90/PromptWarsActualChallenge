## 🎯 What Is This?

**Needy Brain** turns habit-breaking from a chore into an experience. Instead of a passive tracker, users get:

1. **A custom AI persona** (e.g., *"Strict Desi Aunty"*, *"Disappointed IIT Professor"*) generated by Gemini 3.5 Flash based on their specific habits and triggers.
2. **Real-time CBT interventions** — when they hit the SOS button during a craving, their AI persona responds with humor, psychological reframing, and a 60-second behavioral challenge.
3. **A gamified rewards system** — earn coins for completing onboarding and surviving urges; spend them to unlock the "Urge Surfing Vault" (a guided breathing exercise).

## 🔌 API Reference

### `GET /api/health`
Health check for deployment monitoring.
```json
{ "status": "ok", "service": "needy-brain-api" }
```

### `POST /api/auth`
Register or log in a user by phone number.
```json
// Request
{ "phone": "9876543210", "password": "mypassword" }

// Response
{
  "user_id": 1,
  "is_new_user": true,
  "onboarding_complete": false,
  "wallet_balance": 0,
  "vault_unlocked": false
}
```

### `POST /api/onboard`
Submit CBT intake answers → Gemini generates AI persona + 3 interventions.
```json
// Request
{
  "user_id": 1,
  "target_habit": "Late night doomscrolling",
  "habit_triggers": "Lying in bed, boredom",
  "underlying_emotion": "Anxiety",
  "future_motivation": "Sleep better, feel rested"
}

// Response
{
  "user_id": 1,
  "assigned_persona": "Strict Desi Aunty",
  "interventions": ["Do 10 pushups", "Drink cold water", "Read 2 pages of a book"],
  "wallet_balance": 50,
  "vault_unlocked": false
}
```

### `POST /api/intervene`
SOS craving button — Gemini responds in-persona with reframing + challenge.
```json
// Request
{ "user_id": 1, "current_feeling": "Stressed from work, want to scroll" }

// Response
{
  "brain_dialogue": "Arey, again stress is there? Classic you!",
  "cbt_challenge": "Drink one full glass of cold water and pace for 60 seconds.",
  "wallet_balance": 60,
  "vault_unlocked": false
}
```

### `POST /api/unlock-vault`
Spend 50 coins to unlock the Urge Surfing Vault.
```json
// Request
{ "user_id": 1 }

// Response
{ "success": true, "wallet_balance": 10, "vault_unlocked": true, "message": "Vault unlocked!" }
```

## 🚀 Running Locally

### Prerequisites
- Python 3.11+
- Node.js 18+
- A [Gemini API Key](https://aistudio.google.com/app/apikey)

### Backend
```bash
cd backend
python -m venv venv
# Windows:
venv\Scripts\activate
# Mac/Linux:
source venv/bin/activate

pip install -r requirements.txt

# Create .env file
echo GEMINI_API_KEY=your_key_here > .env

uvicorn main:app --reload
# API running at http://localhost:8000
# Swagger docs at http://localhost:8000/docs
```

### Frontend
```bash
cd frontend
npm install
npm run dev
# App running at http://localhost:5173
```

### Run Tests
```bash
cd backend
pytest test_main.py -v
```

## ☁️ Deploy on Render (Manual Steps)

The `render.yaml` Blueprint at the repo root handles infrastructure-as-code. Here's what it deploys:

| Service | Type | Description |
|---------|------|-------------|
| `actual-backend-rm` | Web Service (Python) | FastAPI backend |
| `actual-frontend-rm` | Static Site | React/Vite frontend |

### Step-by-Step Manual Deployment

1. **Push this repo to GitHub** (if not already done)

2. **Go to [render.com](https://render.com)** → New → Blueprint

3. **Connect your GitHub repo** → Render auto-detects `render.yaml`

4. **Set Environment Variables** in the Render dashboard for `actual-backend-rm`:
   | Variable | Value |
   |----------|-------|
   | `GEMINI_API_KEY` | Your Gemini API key from [aistudio.google.com](https://aistudio.google.com) |
   | `FRONTEND_URL` | The URL of your deployed frontend (e.g., `https://actual-frontend-rm.onrender.com`) |
   | `PASSWORD_SALT` | Any random secret string |

5. **Deploy** → Render builds and deploys both services automatically

6. **Test** the deployed API at `https://actual-backend-rm.onrender.com/docs`

> **Note:** On the free tier, services spin down after 15 minutes of inactivity. First request may take ~30 seconds to wake up.

### What `render.yaml` Does
```yaml
services:
  - type: web           # Backend: FastAPI Python service
    name: actual-backend-rm
    env: python
    rootDir: backend
    buildCommand: pip install -r requirements.txt
    startCommand: uvicorn main:app --host 0.0.0.0 --port $PORT
    envVars:
      - key: GEMINI_API_KEY
        sync: false     # Set manually in dashboard (never committed)
  
  - type: static        # Frontend: Vite build → static files
    name: actual-frontend-rm
    rootDir: frontend
    buildCommand: npm install && npm run build
    publishDir: dist
    envVars:
      - key: VITE_API_URL
        fromService:
          type: web
          name: actual-backend-rm
          property: host  # Auto-injected backend URL
```

## 🔐 Security

- **Passwords** are hashed using HMAC-SHA256 (never stored as plaintext)
- **API Keys** are loaded from environment variables only — never in source code
- **`.env` is gitignored** — see `backend/.gitignore`
- **CORS** is restricted to the `FRONTEND_URL` environment variable (no wildcard)
- **Input validation** enforced via Pydantic v2 field validators on all endpoints

## 🧠 AI Prompt Design

Two Gemini prompts power the experience:

**Onboarding Prompt** — Receives habit, triggers, emotion, and motivation. Returns a custom Indian-context persona name + 3 CBT-based behavioral interventions as structured JSON.

**Intervention Prompt** — Receives the user's persona, habit, interventions list, and real-time feeling. Responds in-character with sharp, culturally-aware humor and a 60-second action challenge.

Both prompts enforce `response_mime_type: application/json` for deterministic parsing.
